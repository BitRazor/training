# clarifications.md — product intent (training app)

**Progress = done, not logged.** (2026-06-15)
- Plain English: the "X/15 done" bar counts exercises you tap ✓ on, NOT ones with a number typed in. Logging still feeds the export.
- Technical: `progress()` counts `state.done`; the earlier "logged value = done" model was replaced at user request.

**✓ collapses to declutter.** (2026-06-16) Tapping ✓ shrinks a finished exercise to just its name; reversible.

**Day-1 end order.** (2026-06-16) tib → hip flexion → **Copenhagen Plank** → suitcase carry → wrist curls. Carry runs **before** wrist curls so grip is fresh; wrist curls are the final forearm burn-out. Copenhagen replaced an earlier cable-adduction.

**Warm-ups.** (2026-06-17) Day 1 (5): sled · WGS · inchworm · drop-lunge · balance. Day 2 (5): hindu · quadruped · DB windmills · stick pass-throughs · band pull-aparts. Quadruped is **Day-2 only** (WGS covers t-spine on leg day). Kept BOTH mobility (stick pass-through) AND activation (band pull-aparts). Dropped Reach & Roll (redundant) + Towel internal-rotation (strengthening, not warm-up).

**Removed from the app, user-confirmed.** (2026-06-17) Hide-done toggle; Pre-session checklist — **this also removed the morning-bodyweight field AND the lift-choice toggles** (pulldown/chinup, machine/db, ext/dips → now locked to defaults pulldown/machine/ext); How-to-test section (test phase done). Keep-screen-on pinned to a **fixed top bar** (always visible).

**Gifs.** Source = fitnessprogramer.com (matches existing watermark) or user-supplied; ALWAYS verify the movement visually (`Read` the image). User files may be `.jpg`.
